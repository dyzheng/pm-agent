# DZ-004: 实现 nspin=2 时 init_chg=dm 及 init_chg=hr 功能

## Objective

GitHub Issue #6974。当前 `init_chg=dm`（从密度矩阵初始化电荷密度）不支持 nspin=2（自旋极化）情况。需要重构代码使其在 LCAO 基组下正确读取 spin-up/spin-down 两个分量的密度矩阵并初始化电荷密度。

同时修复 `init_chg=hr`（从哈密顿量矩阵初始化）在 nspin=2 时的支持。nspin=2 时 HR 存储在 `HamiltLCAO::hRS2` 双倍缓冲区中，当前 `init_hr_from_file` 只读取一个文件且 `refresh(false)` 显式拒绝 nspin=2。

### 关键发现

1. **`init_dm_from_file()` 已支持 nspin=2** — 代码已循环读取 dmrs1/dmrs2，问题可能不在读取逻辑本身，而在上游调用链或写出逻辑。需要排查：
   - `before_scf()` 中 `init_chg=="dm"` 分支是否被 nspin=2 条件阻断
   - 密度矩阵写出时是否正确输出两个 CSR 文件
   - `dm2rho()` 对 nspin=2 的积分是否正确

2. **原 spec 遗漏了核心文件 `LCAO_set.cpp`** — 已补全。

3. **`init_chg=hr` 在 nspin=2 时存在两个阻断点**:
   - `init_hr_from_file` 只读取 `hrs1_nao.csr` 一个文件，未读取 `hrs2_nao.csr`
   - `refresh(false)` 在 nspin=2 时直接 `WARNING_QUIT`，程序会退出

> **refresh_times 机制解析**:
> - 正常 SCF: `refresh(true)` 设 `refresh_times=1`，第一次 spin 切换时触发一次 HR 重算（因为 Veff 依赖 spin），之后 `refresh_times=0` 不再重算

## Implementation Guide

### Part A: init_chg=dm 问题定位

由于 `init_dm_from_file()` 已正确处理 nspin=2 的读取逻辑，实际 bug 可能在以下环节：

1. **上游阻断** — `before_scf()` 中 `init_chg=="dm"` 分支是否在 nspin=2 时被跳过或报错
2. **写出缺失** — SCF 收敛后密度矩阵写出逻辑是否在 nspin=2 时正确输出 dmrs1 + dmrs2 两个文件
3. **dm2rho 积分** — `cal_gint_rho(dmr, nspin, rho)` 在 nspin=2 时是否正确遍历两个 spin 分量
4. **DMR 初始化** — `init_DMR()` 在 nspin=2 时是否为两个 `_DMR[0]` 和 `_DMR[1]` 都正确分配了结构

排查步骤:
1. 在 nspin=2 + init_chg=dm 条件下运行，确认是否进入 `init_chg_dm()` (加日志或断点)
2. 检查 `density_matrix_io.cpp` 中 `save_DMR()` 对 nspin=2 的处理
3. 检查 `cal_gint_rho()` 内部对 `dmr` vector 的遍历逻辑
4. 检查 `init_DMR()` 是否为 `_DMR[1]` 正确创建了 HContainer 结构

关键细节:
- k 点索引：nspin=2 时 `nks = 2*nk`，前 nk 个是 spin-up，后 nk 个是 spin-down
- `dm2rho` 中 `dmr[is]` 对应 `rho[is]`
- `init_chg_dm` 调用 `dm2rho` 时 `skip_normalize=true`，归一化由后续 `init_scf` 处理

### Part B: init_chg=hr nspin=2 实现方案

#### 问题分析

nspin=2 + init_chg=hr 存在两个阻断点，需要逐一修复：

**阻断点 1: `before_scf()` 只读取一个 HR 文件** [esolver_ks_lcao.cpp:185]

当前代码硬编码读取 `hrs1_nao.csr` 到 `hR`，但 nspin=2 时：
- `hR` 的底层数据指向 `hRS2` 的 spin-up 半区
- spin-down 半区 (`hRS2[nnr..2nnr-1]`) 未被填充
- 需要额外读取 `hrs2_nao.csr` 到 spin-down 半区

**阻断点 2: `refresh(false)` 拒绝 nspin=2** [hamilt_lcao.cpp:538-542]

`refresh(false)` 的语义是"HR 已从外部加载，不需要算符链重算"。当前 nspin=2 时直接 `WARNING_QUIT`。需要实现 nspin=2 的 `refresh(false)` 逻辑。

> **关键语义**: `refresh(false)` + nspin=2 意味着:
> - `hr_done=true`: 算符链不重算 HR
> - `refresh_times=0`: `updateHk` 中 spin 切换时不触发 HR 重算
> - `current_spin=0`: 从 spin-up 开始
> - hR 指向 hRS2 spin-up 半区
>
> 后续 `HSolverLCAO::solve()` 循环 k 点时，`updateHk(ik)` 会根据 `kv->isk[ik]`
> 切换 hR 指针到对应 spin 半区。由于 `refresh_times=0`，不会触发 `set_hr_done(false)`，
> 算符链只做 folding (HR→HK) 而不重算 HR。这正是从文件加载 HR 后的期望行为。

#### refresh_times 状态机总结

```
                    refresh(true)              refresh(false)
                    (正常 SCF)                  (HR 从文件加载)
                    ┌──────────┐               ┌──────────┐
                    │hr_done=F │               │hr_done=T │
                    │ref_times=1│               │ref_times=0│
                    │cur_spin=0 │               │cur_spin=0 │
                    └─────┬────┘               └─────┬────┘
                          │                          │
            updateHk(ik), spin 切换                updateHk(ik), spin 切换
                          │                          │
                    ┌─────▼────┐               ┌─────▼────┐
                    │ref_times>0│               │ref_times=0│
                    │→ ref_times--│              │→ 不触发重算│
                    │→ hr_done=F │              │→ 只切换指针│
                    │→ 触发HR重算│              └──────────┘
                    └─────┬────┘
                          │
                    第二次 spin 切换
                          │
                    ┌─────▼────┐
                    │ref_times=0│
                    │→ 不再重算 │
                    └──────────┘
```

> 正常 SCF 中 `refresh_times=1` 的含义: 第一次 spin 切换时需要重算一次 HR（因为 Veff 依赖 spin），
> 之后两个 spin 的 HR 都已计算并存储在 hRS2 中，后续切换只需移动指针。
>
> init_chg=hr 中 `refresh_times=0` 的含义: HR 已从文件完整加载（两个 spin 都有），
> 任何 spin 切换都只需移动指针，永远不需要重算。
